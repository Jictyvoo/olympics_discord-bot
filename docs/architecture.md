# Architecture

## What this is

`olhojogo` вҖ” eye on the game (PT *olho* + *jogo*) вҖ” is a multi-provider sports event monitoring
daemon. It pulls schedule and result data from upstream sports APIs (Olympics SPH, FIFA, вҖҰ),
persists them to a local SQL store (SQLite or MySQL), and emits Discord notifications and
scheduled events.

## Key design decisions

**Generic domain model.** `internal/domain/eventcore/` has no Olympics-specific types. `Fixture`,
`Participant`, `Result` are provider-neutral. Fetcher mappers translate API responses into
`eventcore` types at the boundary.

**Repositories vs fetchers.** Two kinds of "repository" historically blur into one. We split:

- `internal/infra/repositories/` вҖ” read+write DB packages (`reposqlite`, `repomysql`).
- `internal/infra/fetchers/` вҖ” read-only HTTP packages that pull data from upstream APIs
  (`olympicsfetch`, `fifafetch`).

**Bootstrap is the only switchboard.** The dialect choice (sqlite vs mysql) and the enabled
fetcher set live in `internal/bootstrap/injections.go`. No `if driver == "mysql"` inside any
`repo*/di.go`; no `if code == "fifa"` inside any `*fetch/di.go`.

**Canonical IDs.** 16-byte deterministic IDs from SHA-256(providerID + 0x00 + externalKey),
stored as 32-char lowercase hex. Re-sync is idempotent; provider namespaces cannot collide.

**Consumer-defined interfaces.** Each use-case package declares the smallest interface it needs
from the store and infra layers. No `ports/` or `gateways/` packages. remy resolves these by
structural duck-typing because both the concrete repo and the interface alias are registered.

**Weak observer registry.** `notifier.observerSet` uses `weak.Pointer[Observer]` so GC'd
observers auto-deregister. Long-lived observers (e.g. `discordsync`) hold a strong reference
through the injector to stay alive.

**No Atlas at runtime.** Migrations are embedded SQL files (generated by Atlas at build time)
applied by `internal/migrator/` (~120 LOC, no third-party runtime). Triggered by
`OLH_DB_MIGRATE=true`.

**CGO-free default.** `net/http` replaces `andelf/go-curl` as the default outbound HTTP client.
The curl path is preserved behind `//go:build curl` for environments that require its TLS quirks.

## Package responsibilities

| Package                                  | Responsibility                                                    |
|------------------------------------------|-------------------------------------------------------------------|
| `cmd/olhojogo`                           | Main daemon entrypoint; subcommands serve/migrate/version         |

| `config`                                 | Typed Config + env binders                                        |
| `internal/bootstrap`                     | DI composition root; dialect + fetcher selection; observer wiring |

| `internal/migrator`                      | Embedded SQL migration runner                                     |
| `pkg/idgen`                              | CanonicalID generation                                            |
| `pkg/confloader`                         | TOML + env config loader                                          |
| `build/entschema`                        | ent schema source of truth (own go.mod)                           |
| `build/migrations/{sqlite,mysql}`        | Atlas-generated migrations, embedded                              |
| `tools/seedgen`                          | Generate idempotent seed SQL from `build/seed/countries.json`     |
