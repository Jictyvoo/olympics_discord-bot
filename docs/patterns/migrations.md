# Migrations

Schema source of truth: ent code in `build/entschema/`. Atlas generates dialect-specific SQL
files into `build/migrations/{sqlite,mysql}/`. The binary embeds them via `//go:embed` and
applies them with the in-house `internal/migrator/`.

## Pipeline

```
build/entschema/*.go
       │
       ▼  atlas migrate diff
build/migrations/sqlite/*.sql      ← embedded via //go:embed in same dir
build/migrations/mysql/*.sql       ← embedded via //go:embed in same dir
       │
       ▼  internal/migrator.Run(ctx)
schema_migrations table
```

Atlas runs at build time only. The runtime applier is the in-house migrator (~120 LOC) — no
`golang-migrate/migrate` in the runtime tree.

## Generating a migration

```bash
task build:migration:gen DIFF_NAME=add_fixtures_checksum
```

Atlas diffs the current ent schema against the latest migration on disk and writes a new
`<timestamp>_<DIFF_NAME>.sql` into the SQLite migration dir. Hash the directory afterwards:

```bash
task build:migration:hash
```

For MySQL: same flow against a MySQL dev URL. Both dialects are committed.

## Runtime application

`olhojogo serve` honours `OLH_DB_MIGRATE=true` (or `--migrate`) and runs the embedded migrations
before the sync loop starts:

```go
m := migrator.New(db, conf.Database.Driver, sqlite.FS)
if err := m.Run(ctx); err != nil { return err }
```

The migrator:

1. Creates `schema_migrations(version VARCHAR PRIMARY KEY, applied_at DATETIME)` on first run
   using the dialect-specific bootstrap statement.
2. Lists `*.sql` files in lexicographic order.
3. Skips files whose `version` is already in `schema_migrations`.
4. Applies the rest inside a transaction per file, recording the version on success.

Multi-statement files require `?multiStatements=true` on the MySQL DSN. SQLite (`modernc.org/sqlite`)
supports multi-statement natively.

## Down migrations

Not supported by the runtime applier. Rollbacks are new forward migrations. This matches how the
project is operated and keeps the runtime simple.

## Seed migrations

Seed data (e.g. countries) is generated by `tools/seedgen` from `build/seed/countries.json`. The
output is a regular timestamped migration in both dialect directories using `INSERT OR IGNORE`
(SQLite) / `INSERT IGNORE` (MySQL) so it's idempotent.

Regenerate with:

```bash
task gen:seed
```

The generated SQL is byte-identical for the same input, so committing it produces stable diffs.

## Forbidden

- Hand-editing files under `build/migrations/{sqlite,mysql}/` after Atlas has written them. Use a
  new migration to amend.
- Skipping `task build:migration:hash` after a manual edit — the Atlas dir hash will mismatch.
- Calling Atlas at runtime. Atlas is a build-time tool only.
- Running migrations from a non-`bootstrap` site.
